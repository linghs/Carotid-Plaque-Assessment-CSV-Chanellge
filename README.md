# Two-Stage 4-Part Model for Carotid Plaque Segmentation and Vulnerability Assessment

A two-stage framework for the **CSV 2026 Challenge**: carotid artery plaque segmentation (3-class) and vulnerability classification (binary). The method decomposes the problem into a segmentation stage and a region-aware classification stage, where 4 masked image parts (plaque/vessel × longitudinal/transverse) are independently encoded and fused for risk prediction.

## Method Overview

```
                        ┌─────────────────────────────────┐
                        │       Input: Ultrasound H5       │
                        │  (long_img + trans_img, 512×512) │
                        └──────────────┬──────────────────┘
                                       │
                        ┌──────────────▼──────────────────┐
                        │    Stage 1: Segmentation Model   │
                        │  FPN + EfficientNet-B4 encoder   │
                        │  Semi-supervised (pseudo labels)  │
                        └──────────────┬──────────────────┘
                                       │
                          long_mask + trans_mask
                          {0: bg, 128: plaque, 255: vessel}
                                       │
                        ┌──────────────▼──────────────────┐
                        │  Stage 2: 4-Part Classification  │
                        │  4 independent ResNet-152 encoders│
                        │                                   │
                        │  Part 1: long_img × mask_128      │
                        │  Part 2: long_img × mask_255      │
                        │  Part 3: trans_img × mask_128     │
                        │  Part 4: trans_img × mask_255     │
                        │                                   │
                        │  → concat fusion → FC → class     │
                        └──────────────┬──────────────────┘
                                       │
                              cls ∈ {0: low, 1: high}
```

**Key design choices:**

1. **Semi-supervised segmentation**: Trains on 200 labeled cases + 800 pseudo-labeled cases generated by ensemble voting across multiple backbone architectures.
2. **Predicted-mask training**: The classification model is trained on masks produced by the segmentation model (not ground truth), aligning the train/test distribution.
3. **Region-aware 4-part input**: Each of the 4 masked regions (plaque/vessel × long/trans) goes through a dedicated encoder, preserving spatial information specific to each anatomical structure.
4. **Optional mask dilation**: Dilating the binary mask before masking allows the model to see contextual pixels around the segmented regions.

## Data Format

The challenge provides data in HDF5 format:

```
data/
├── train/
│   ├── images/          # 1000 cases: 0000.h5 ... 0999.h5
│   │   └── *.h5         # Keys: 'long_img', 'trans_img' (512×512 grayscale)
│   └── labels/          # 200 labeled cases: 0000_label.h5 ... 0199_label.h5
│       └── *.h5         # Keys: 'long_mask', 'trans_mask' (0/128/255), 'cls' (0/1)
├── val/
│   └── images/          # Validation cases
│       └── *.h5
└── pseudo_labels/       # (Generated) pseudo labels for cases 200-999
    └── *.h5
```

**Segmentation classes**: `0` = background, `128` = plaque, `255` = vessel

**Classification**: `0` = low risk, `1` = high risk

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Train (full pipeline)

```bash
bash scripts/train_all.sh
```

This runs three stages sequentially:
1. Semi-supervised segmentation training → `checkpoints/seg_semi/best_model.pth`
2. Generate seg predictions on training data → `checkpoints/seg_semi/preds/`
3. 4-part classification training → `checkpoints/cls_4parts/best_model.pth`

### 3. Predict

```bash
bash scripts/predict.sh
```

Produces `data/val/preds.tar.gz` ready for submission.

## Training Details

### Stage 1: Segmentation

| Setting | Value |
|---------|-------|
| Architecture | FPN + EfficientNet-B4 |
| Loss | Dice + CrossEntropy (0.5 + 0.5) |
| Optimizer | AdamW (lr=1e-4, wd=1e-4) |
| Scheduler | CosineAnnealing |
| Image size | 512×512 |
| Augmentation | HFlip, BrightnessContrast, GaussNoise, ShiftScaleRotate |
| Semi-supervised | 200 real + 800 pseudo-labeled cases |

### Stage 2: 4-Part Classification

| Setting | Value |
|---------|-------|
| Architecture | 4× ResNet-152 encoders + concat fusion + FC |
| Loss | CrossEntropy (with class weights) |
| Optimizer | AdamW (lr=1e-4) |
| Scheduler | CosineAnnealing |
| Input | 4 masked image regions (plaque/vessel × long/trans) |
| Dilation | Configurable (default: 0, tested 0/5/10/15) |

### Evaluation Metrics

Following the challenge protocol:
- **Segmentation Score**: `S_seg = 0.5 × S_long + 0.5 × S_trans`, where `S_v = 0.6 × S_plaque + 0.4 × S_vessel` and each `S = (DSC + NSD) / 2`
- **Classification Score**: `S_cls = mean(F1_per_class)`

## Step-by-Step Guide

### Training individual stages

```bash
# Stage 1A: Supervised segmentation only (no pseudo labels)
bash scripts/train_seg.sh

# Stage 1B: Semi-supervised segmentation (recommended)
bash scripts/train_seg_semi.sh

# Stage 2: Generate segmentation predictions on training set
bash scripts/generate_seg_preds.sh

# Stage 3: Train 4-part classification model
bash scripts/train_cls_4parts.sh
```

### Generating pseudo labels (optional)

If you have multiple trained segmentation models with different backbones, you can generate pseudo labels via ensemble voting:

```bash
bash scripts/generate_pseudo_labels.sh
```

This requires model checkpoints in directories like `csv_seg_outputs_<encoder>/best_model.pth`.

### Custom configuration

All scripts expose configuration variables at the top. Key parameters:

```bash
# In scripts/train_cls_4parts.sh
ENCODER="resnet152"          # Backbone for classification
FUSION_METHOD="concat"       # concat | add | attention
DILATION_KERNEL_SIZE=0       # 0 = no dilation, 5/10/15 = dilation radius
```
